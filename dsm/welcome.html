Introduction

Welcome to the IBM Distributed State Machine Toolkit (DSMtk). The library provides a set of Java APIs and tools to create and manage distribute state machine whose state is defined through a set of declarative rules.

Prerequisites

  • Java 1.6 or later JRE
  • Windows XP or Linux
  • Apache and Antlr 3 components.

Installation/Configuration

Installation is the same for Windows and *nix systems and basically entails unzipping the release, setting some environment variables, and adding a bin directory to the execution search path.
  1. Choose an install directory and create it. For example /opt/dsm. mkdir /opt/dsm.
  2. Set your DSM_HOME environment variable to the directory created above.
    • On Linux - export DSM_HOME=/opt/dsm
    • On Windows - set DSM_HOME=c:\opt\dsm
  3. Unzip the distribution. cd /opt/dsm. unzip dsm-X.X-release.zip.
  4. Add /opt/dsm/bin to your execution PATH env var
  5. If needed (i.e. not included in this DSM distribution), download and install the software listed below into the lib directory of the DSM installation directory tree.
    Component Files Location
    Apache Derby 10.8 derby.jar, derbyclient.jar http://db.apache.org/derby/releases/release-10.8.1.2.html
    Apache Commons Lang 2.6 commons-lang-2.6.jar http://commons.apache.org/lang/download_lang.cgi
    Apache Commons Logging 1.1.1 commons-logging-1.1.1.jar http://commons.apache.org/logging/download_logging.cgi
    Apache JEXL 2.1.1 commons-jexl-2.1.1.jar http://commons.apache.org/jexl/download_jexl.cgi
    Antlr 3.4 antlr-3.4-complete.jar http://www.antlr.org/download.html

Contents

The release is organized as follows:
  1. doc directory - contains written documentation and javadoc.
  2. bin directory - contains tools.
  3. samples directory - contains various Java samples demonstrating the use of the DSMtk APIs.
  4. lib directory - jars, properties files, etc.
  5. license directory - licenses for software components contained in the DSMtk package.
